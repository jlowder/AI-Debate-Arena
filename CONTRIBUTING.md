# Contributing to AI Debate System

Thank you for your interest in contributing to the AI Debate System! This document provides guidelines for contributing to this project.

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Test your changes thoroughly
5. Submit a pull request

## Development Setup

1. Install Python 3.7+
2. Install Ollama and set up a model
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## Code Style

- Follow PEP 8 style guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and avoid excessive complexity

## Testing

Before submitting a pull request:

1. Test your changes manually
2. Verify that existing functionality still works
3. Update the README.md if you've added new features or changed behavior

## Pull Request Process

1. Update the README.md with details of changes to the interface
2. Add an appropriate version tag using semantic versioning (major.minor.patch)
3. Ensure all tests pass
4. Your pull request will be reviewed by maintainers
